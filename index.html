The Artifact: iOS Inventory Management App

Originally created in CS 360: Mobile Architecture and Programming, this application was enhanced across three categories for the capstone.

iOS Inventory Management Application

A comprehensive iOS inventory management application built with Swift and SwiftUI that allows businesses to track inventory across multiple locations, manage suppliers, handle user authentication with role-based access control, and generate reports.

Key Features

  • User authentication with secure password hashing
  • Biometric login (Face ID / Touch ID)
  • Role-based access control (Admin, Manager, Staff, Viewer)
  • Multi-location inventory tracking
  • Barcode / QR code scanning
  • Real-time search and filtering
  • Category & supplier management
  • Stock transfer between branches
  • Comprehensive audit logging
  • Data backup, restore & export (CSV, JSON, PDF)
  • Dashboard with analytics and low stock alerts

Technologies Used

Swift 5 SwiftUI SwiftData AVFoundation LocalAuthentication CryptoKit MVVM Architecture

Course Outcomes

This ePortfolio demonstrates mastery across all five CS 499 course outcomes.

  • 1
    Collaborative Environments

    Employ strategies for building collaborative environments that enable diverse audiences to support organizational decision making in the field of computer science.

  • 2
    Professional Communication

    Design, develop, and deliver professional-quality oral, written, and visual communications that are coherent, technically sound, and appropriately adapted to specific audiences and contexts.

  • 3
    Algorithmic Principles

    Design and evaluate computing solutions that solve a given problem using algorithmic principles and computer science practices and standards appropriate to its solution, while managing the trade-offs involved in design choices.

  • 4
    Innovative Techniques

    Demonstrate an ability to use well-founded and innovative techniques, skills, and tools in computing practices for the purpose of implementing computer solutions that deliver value and accomplish industry-specific goals.

  • 5
    Security Mindset

    Develop a security mindset that anticipates adversarial exploits in software architecture and designs to expose potential vulnerabilities, mitigate design flaws, and ensure privacy and enhanced security of data and resources.
